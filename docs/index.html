‚öôÔ∏è Configuration

Configure your environment variables to connect all services:

1Create Environment File

Create a .env.local file in the root directory:

touch .env.local

2Add Environment Variables

Add the following variables to your .env.local file:

# ============================================ # DATABASE # ============================================ DATABASE_URL=your_neon_database_url_here # ============================================ # CLERK AUTHENTICATION # ============================================ NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_xxxxx CLERK_SECRET_KEY=sk_test_xxxxx # Clerk Redirect URLs (Development) NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/home NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/home # ============================================ # ADMIN PANEL ACCESS # ============================================ ADMIN_USER_ID=your_clerk_user_id_here # For multiple admins, separate with commas: # ADMIN_USER_ID=user_id_1,user_id_2,user_id_3 # ============================================ # STRIPE PAYMENT (Optional) # ============================================ STRIPE_API_SECRET_KEY=sk_test_xxxxx NEXT_PUBLIC_APP_URL=http://localhost:3000 # Stripe Webhook (for production) STRIPE_WEBHOOK_SECRET=whsec_xxxxx

Getting Your API Keys

1. Neon Database URL

  1. Go to console.neon.tech
  2. Create a new project or select existing one
  3. Navigate to "Connection Details"
  4. Copy the connection string (pooled connection recommended)
  5. Paste it as your DATABASE_URL

2. Clerk Authentication Keys

  1. Go to dashboard.clerk.com
  2. Create a new application or select existing one
  3. Navigate to "API Keys" in the sidebar
  4. Copy the "Publishable Key" and "Secret Key"
  5. Paste them into your environment file

3. Admin User ID

  1. Start your development server (see Usage Guide)
  2. Sign up or sign in to your application
  3. Go back to Clerk Dashboard ‚Üí Users
  4. Click on your user account
  5. Copy the "User ID" from the user details
  6. Add it to ADMIN_USER_ID in your environment file

4. Stripe Keys (Optional)

  1. Go to dashboard.stripe.com
  2. Navigate to "Developers" ‚Üí "API keys"
  3. Use test keys for development
  4. Copy the "Secret key" and paste it as STRIPE_API_SECRET_KEY
üîí Security Warning: Never share or publicly expose your .env.local file. Keep it secure and private at all times.

üóÑÔ∏è Database Setup

Set up and populate your database with the following steps:

1Push Database Schema

Create all necessary tables in your database:

npm run db:push # Or with npx npx drizzle-kit push

This command will create the following tables:

  • courses - Language courses
  • units - Course units
  • lessons - Individual lessons
  • challenges - Learning challenges
  • challenge_options - Answer options
  • challenge_progress - User progress tracking
  • user_progress - Overall user progress
  • user_subscription - Subscription management

2Seed Database

Populate your database with sample data (courses, lessons, challenges):

npm run db:seed

This will add:

  • 4 Language courses (Spanish, Italian, French, Croatian)
  • Sample units and lessons
  • Interactive challenges with audio
  • Challenge options (correct and incorrect answers)

3View Database (Optional)

Open Drizzle Studio to view and manage your database:

npm run db:studio

This will open a visual database browser at https://local.drizzle.studio

üí° Note: The seed script includes sample Spanish lessons. You can modify src/server/scripts/seed.ts to add your own content.

üîß Troubleshooting

Common Issues

Database Connection Error

Error: "Failed to connect to database"

Solution:

  • Verify DATABASE_URL is correct
  • Ensure your Neon database is active
  • Check if you're using the pooled connection URL
  • Test connection with: npx drizzle-kit push

No Courses Showing

Error: Empty course page

Solution:

  • Run seed script: npm run db:seed
  • Check database has data: npm run db:studio
  • Refresh your browser

Admin Panel Not Accessible

Error: Redirected from /admin to home

Solution:

  • Ensure ADMIN_USER_ID is set in .env.local
  • Verify you're using the correct User ID from Clerk
  • Restart your development server
  • Sign out and sign back in
